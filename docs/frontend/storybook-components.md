# Візуальна документація (Storybook)

Цей розділ описує стани ключових UI-компонентів frontend-застосунку для управління замовленнями, згідно з методологією Component-Driven Development.

## Компонент: CheckoutButton (Кнопка оформлення замовлення)

**Опис:** Основна кнопка (Call to Action) у кошику користувача, яка ініціює запит на створення замовлення (`POST /api/orders`).

### Сценарії (Stories)

1. **Default (Базовий стан)**
   * **Вигляд:** Синя кнопка з текстом "Оформити замовлення".
   * **Поведінка:** Активна, при кліку викликає функцію сабміту форми.

2. **Disabled (Неактивний стан)**
   * **Умова:** Кошик порожній або є невалідні дані у формі доставки.
   * **Вигляд:** Сіра кнопка, зменшена контрастність тексту.
   * **Поведінка:** Кліки ігноруються.

3. **Loading (Стан завантаження)**
   * **Умова:** Відправлено запит на бекенд, очікується відповідь.
   * **Вигляд:** Текст замінюється на анімацію спінера (завантаження).
   * **Поведінка:** Кнопка блокується для запобігання повторному кліку та дублюванню замовлень.
