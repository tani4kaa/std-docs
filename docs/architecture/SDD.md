# Software Design Document (SDD)

## 1. Загальна архітектура системи
Для реалізації веб-застосунку обрано клієнт-серверну архітектуру (монолітно-модульний підхід), що забезпечує чіткий поділ між інтерфейсом користувача та бізнес-логікою.

## 2. Перелік та опис компонентів
* **Frontend:** реалізовано на базі бібліотеки React.js як Single Page Application (SPA). Відповідає за рендеринг інтерфейсу, валідацію форм на стороні клієнта та взаємодію з API.
* **Backend:** побудовано на платформі Node.js з використанням фреймворку Express. Забезпечує обробку запитів, аутентифікацію користувачів та виконання операцій з базою даних.
* **Database:** використовується реляційна СУБД PostgreSQL. Вона забезпечує цілісність даних та зберігає інформацію про профілі користувачів, каталог товарів та історію замовлень.

## 3. Опис API
Взаємодія між Frontend та Backend відбувається за допомогою протоколу HTTP з використанням стандартів REST API.

| Метод | Ендпоінт | Опис функціональності |
| :--- | :--- | :--- |
| **GET** | `/api/orders` | Отримання списку замовлень авторизованого користувача. |
| **POST** | `/api/orders` | Створення нового замовлення на основі даних з кошика. |
| **PUT** | `/api/orders/{id}` | Оновлення статусу замовлення (для адміністраторів). |
| **GET** | `/api/products` | Отримання каталогу доступних товарів. |
