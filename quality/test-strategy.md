# Якість та Матриця простежуваності (Test Strategy)

Даний документ визначає загальний підхід до забезпечення якості веб-застосунку для управління замовленнями.

## 1. Рівні тестування
* **Unit Testing:** перевірка окремих функцій Backend-частини.
* **Integration Testing:** перевірка коректності взаємодії між мікросервісами та БД через REST API.
* **System / Acceptance Testing:** комплексна перевірка всього ланцюга: від реєстрації клієнта до успішної оплати замовлення.

## 2. Інструментарій
* **Jest / Mocha:** для автоматизації Unit-тестів.
* **Postman / Newman:** для тестування API-ендпоінтів.
* **Selenium / Playwright:** для автоматизації UI-сценаріїв у браузері.

## 3. Матриця простежуваності (Traceability Matrix)
Усі функціональні вимоги беруться з головного джерела істини: [Документа SSD](../architecture/SSD.md).

| ID Вимоги | Опис вимоги | ID Тесту | Результат |
| :--- | :--- | :--- | :--- |
| **FR-01** | Реєстрація та авторизація | **TC-01** | Перевірка успішного входу з валідними даними. |
| **FR-02** | Створення замовлення | **TC-02** | Перевірка коректності суми при додаванні товарів. |
| **FR-03** | Зміна статусу замовлення | **TC-03** | Перевірка оновлення статусу в БД після дій менеджера. |
