# System Specification Document (SSD)

## 1. Призначення та межі системи
Система призначена для автоматизації процесу продажу товарів через веб-інтерфейс. Вона охоплює шлях користувача від авторизації до отримання підтвердження замовлення та дозволяє персоналу керувати життєвим циклом покупок. Межі системи включають клієнтську частину (браузер), сервер розробки та базу даних замовлень.

## 2. Основні функції (Functional Requirements)
* **FR-01: Реєстрація та авторизація.** Система повинна підтримувати створення облікових записів користувачів та забезпечувати безпечний вхід за допомогою логіна і пароля.
* **FR-02: Створення замовлення.** Користувач має можливість обирати товари з каталогу, додавати їх до кошика та підтверджувати покупку.
* **FR-03: Управління статусами.** Сценарій для адміністратора, що дозволяє змінювати стан замовлення (наприклад, з "Очікується" на "Відправлено") для інформування клієнта.

## 3. Обмеження та припущення
* Система вимагає стабільного інтернет-з'єднання для взаємодії з API.
* Оплата товарів на даному етапі вважається успішною за замовчуванням (припущення для MVP).
* Максимальна кількість товарів в одному замовленні не повинна перевищувати 50 одиниць.

## 4. Нефункціональні вимоги
* **Продуктивність:** Система повинна обробляти не менше 500 запитів на хвилину з середнім часом відповіді до 300 мс.
* **Доступність:** Показник доступності сервісу має бути не менше 99.9% часу на рік.
* **Безпека:** Всі паролі користувачів повинні зберігатися у базі даних у хешованому вигляді; використання HTTPS є обов’язковим.
